# 项目需求文档：AI 双语 EPUB 阅读器浏览器插件

## 1. 项目定位

将现有项目调整为一个基于浏览器插件的 EPUB 阅读器，运行形态以 Chrome/Chromium 浏览器插件为主，不再提供独立 Web 站点、PWA、后台管理或用户账号体系。

插件目标是让用户在浏览器侧边栏中完成以下动作：

1. 导入本地 EPUB 文件或通过 URL 拉取 EPUB。
2. 在本地书架中管理已导入书籍。
3. 阅读章节内容，并进行段落双语翻译、摘要、划词/短语翻译。
4. 在插件内配置任意 OpenAI 兼容大模型服务。

## 2. 本次版本范围

### 2.1 保留能力

1. EPUB 解析与目录抽取。
2. 本地书架存储。
3. 章节渲染与目录跳转。
4. AI 翻译、AI 摘要、选词/短语翻译。
5. 多语言界面基础能力。

### 2.2 删除能力

1. 用户登录。
2. 手机验证码。
3. 会员订阅与支付。
4. 推荐邀请。
5. 后台管理系统。
6. 所有依赖业务后端才可运行的核心流程。

## 3. 核心产品要求

### 3.1 插件形态

1. 使用 Chrome Extension Manifest V3。
2. 插件主界面运行在 `side_panel` 中。
3. 点击插件图标时自动打开侧边栏。
4. 所有核心功能应可在插件本地完成，不依赖登录态。

### 3.2 阅读与导入

1. 支持拖拽或选择本地 `.epub` 文件导入。
2. 支持通过外部 URL 导入 EPUB。
3. 已导入图书保存到 IndexedDB，本地持久化。
4. 支持书籍打开、删除、重新下载。

### 3.3 AI 能力

1. 所有 AI 能力对本地用户直接可用，不设会员门槛。
2. 支持整章摘要。
3. 支持段落流式双语翻译。
4. 支持选词或短语翻译。
5. 流式响应过程中需要实时更新 UI。

### 3.4 大模型配置

插件设置中必须支持本地配置以下内容：

1. Provider 类型：`openai`、`deepseek`、`custom`。
2. `Base URL`。
3. `API Key`。
4. `Model`。
5. 翻译系统提示词。
6. 摘要系统提示词。

要求：

1. 配置保存在本地存储中。
2. 不上传到业务服务器。
3. 应允许用户接入任意 OpenAI 兼容接口。
4. 默认提供可编辑的提示词模板。

## 4. 非功能要求

### 4.1 隐私

1. 书籍文件只保存在用户本地。
2. API Key 只保存在用户本地。
3. 不引入账号系统。

### 4.2 安全

1. 章节 HTML 渲染前必须清洗。
2. 优先使用 iframe 或等效隔离方案。
3. 插件权限控制在最小可用范围。

### 4.3 可维护性

1. 继续保留核心解析、渲染、LLM、存储分层。
2. 平台相关逻辑集中在 `adapters/` 或插件入口层。
3. 旧的登录、支付、后台逻辑不再进入主运行路径。

## 5. 技术实现要求

### 5.1 技术栈

1. React
2. TypeScript
3. Vite
4. Zustand
5. IndexedDB / Dexie
6. Chrome Extension MV3

### 5.2 插件清单

需要新增或完善：

1. `public/manifest.json`
2. `public/background.js`
3. `side_panel.default_path = "index.html"`
4. `permissions` 至少包含 `storage`、`sidePanel`
5. `host_permissions` 允许访问用户配置的大模型域名以及远程 EPUB 链接

## 6. 开发任务拆分

### Phase 1：需求与运行路径收口

1. 将产品定位统一为浏览器插件。
2. 去掉登录、支付、邀请、后台入口。
3. 设置页改为本地模型配置中心。

### Phase 2：插件化最小可运行版本

1. 补齐 Manifest V3。
2. 提供 background service worker。
3. 确保 `npm run build` 后产物可作为插件加载。

### Phase 3：AI 直连能力

1. LLM 请求不再依赖业务后端代理。
2. 按用户本地配置直接请求 OpenAI 兼容接口。
3. 保留流式解析和重试机制。

### Phase 4：阅读体验补强

1. 远程 EPUB 导入兼容插件环境。
2. 打磨设置、导入、阅读、翻译的主流程。
3. 清理旧模块残留引用。

## 7. 验收标准

满足以下条件视为本阶段完成：

1. 项目需求文档已切换为“浏览器插件 + 本地模型配置”。
2. 主界面无需登录即可进入书架与阅读器。
3. 设置面板可保存并读取模型配置。
4. 翻译和摘要使用本地模型配置发起请求。
5. 项目可构建出可加载的浏览器插件产物。

## 8. 当前开发决策

本轮开发按以下假设推进：

1. 以 Chrome/Chromium 插件为第一目标平台。
2. 用户自行提供 API Key 与模型服务地址。
3. 旧的后台相关代码暂不全部删除，但不再进入主流程。
